# Berkontribusi

Terima kasih sudah tertarik membantu. Project ini sepenuhnya komunitas — siapapun boleh ikut.

## Cara Cepat Berkontribusi

### 1. Lapor kesalahan

Nemu typo, kalimat aneh, kode yang error, atau penjelasan yang membingungkan? Buka [Issue baru](#) dengan info:

- Bab dan bagian mana
- Apa yang salah
- (Kalau bisa) saran perbaikan

### 2. Sarankan perbaikan

Punya analogi yang lebih bagus? Contoh yang lebih relevan untuk pembaca Indonesia? Buka Issue dengan label `improvement`.

### 3. Edit langsung (Pull Request)

Untuk perubahan kecil (typo, kalimat), edit langsung lewat tombol pensil di GitHub. Untuk perubahan besar (rewrite bab, tambah bagian), buka Issue dulu untuk diskusi sebelum kerja.

## Pedoman Gaya Bahasa

Project ini punya satu prinsip utama: **pemula total harus bisa paham**.

### Yang dianjurkan

- Pakai "kamu" untuk menyapa pembaca, kecuali konteksnya formal
- Kalimat pendek lebih baik dari kalimat panjang berlapis
- Setiap konsep baru: kasih analogi dunia nyata
- Setiap istilah teknis: jelaskan saat pertama kali muncul, walaupun nanti ditulis dalam Inggris saja
- Contoh kode: pakai nama dan konteks Indonesia (`siswa`, `harga_kopi`, `daftar_belanja` — bukan `foo`, `bar`)

### Yang dihindari

- Terjemahan literal yang janggal — utamakan makna, bukan kata
- Istilah teknis yang dipaksa di-Indonesiakan kalau ternyata jadi aneh ("perulangan" untuk "loop" boleh, tapi "lingkaran" untuk "loop" jangan)
- Asumsi pembaca sudah tahu — kalau ragu, jelasin
- Humor atau referensi yang hanya nyambung untuk satu generasi/daerah

### Istilah Teknis: Inggris atau Indonesia?

Aturannya: **pakai yang paling sering dipakai programmer Indonesia di kehidupan nyata**.

| Istilah Inggris | Tetap Inggris? | Catatan |
|---|---|---|
| variable | Ya | "variabel" juga oke |
| function | Ya | "fungsi" juga oke, sering dipakai |
| loop | Ya | "perulangan" sebagai padanan saat menjelaskan |
| string | Ya | "teks" sebagai padanan |
| integer | Ya | "bilangan bulat" sebagai padanan |
| list | Ya | jangan diterjemahkan jadi "daftar" di kode |
| dictionary | Ya | "kamus" sebagai padanan saat menjelaskan |
| boolean | Ya | "boolean" — biarkan |
| if/else | Ya | jangan diterjemahkan |
| return | Ya | "mengembalikan" sebagai kata kerja |
| error | Ya | "kesalahan" juga oke |
| install | Boleh "instal" | sudah baku di KBBI |
| terminal | Ya | sudah dipakai luas |

Saat istilah Inggris pertama kali muncul, kasih penjelasan singkat: _"variable (variabel — tempat menyimpan data)"_.

## Pedoman Teknis

### Format File

- Markdown (.md), encoding UTF-8
- Heading H1 hanya satu per file (judul bab)
- Code block selalu pakai language tag: `` ```python `` bukan ` ``` `

### Struktur Bab

Tiap bab idealnya punya:

1. **Pembukaan** — kenapa bab ini penting, kepakai untuk apa
2. **Konsep** — penjelasan teori dengan analogi
3. **Contoh kode** — kode kecil yang bisa langsung dijalankan
4. **Project mini** (untuk bab Bagian 2) — gabungan semua konsep
5. **Ringkasan** — poin utama dalam 5-7 bullet
6. **Latihan** — soal untuk dikerjakan pembaca

### Jangan Lupa Atribusi

Setiap halaman bab harus include footer kecil:

```
---
Diterjemahkan dari [Chapter X](https://automatetheboringstuff.com/2e/chapterX/) karya Al Sweigart, dilisensikan CC BY-NC-SA.
```

## Code of Conduct Singkat

- Hormati kontributor lain. Beda pendapat boleh, ngeremehin tidak.
- Pemula adalah audience utama — pertanyaan dasar tidak pernah bodoh.
- Tidak ada self-promotion produk berbayar di konten buku.

Itu saja. Selamat berkontribusi.
